# Notes
#
# • input: When JSON Probe is true, the script binds the driver and points input at the kernel
#   sysfs raw attribute for the wired hardware channel: in_voltage<Id-1>_raw
#   (defaults to in_voltage<j-1>_raw when no channel map is given). Paths differ by SoC/driver.
#
# • Channels[] (preferred, per Device): array of {Id, Type}, one per logical channel. Id (1-based)
#   is the hardware A2D input the sensor is wired to (holes allowed); Type (rop/flex/embedded)
#   selects that channel's thresholds from the device Types[] array, so two channels on one chip
#   can carry different bounds. A multi-entry Channels[] means one present chip serves all NumChnl
#   channels (BOM-alternative), and each channel directory is named by its mapped Id — so
#   Channels [{Id:1},{Id:4}] produces .../<N>/1/ and .../<N>/4/ (not 1/ and 2/). Id drives the input
#   symlink (in_voltage<Id-1>_raw), the ADS1015 MUX byte (0xc2/0xd2/0xe2/0xf2 = inputs 1-4) and the
#   MAX1363 scan/monitor bytes in CfgRegVal. Legacy scalar/array ChannelId + device-level Type are
#   still honored when Channels[]/Types[] are absent. See hw-management-bmc-a2d-leakage-config-example.json.
#
# • Validate on a live BMC with bmc/tests/hw-management-bmc-leakage-validate.sh: a minimal HW-vs-SW
#   test that prints each configured Bus/Address I2C probe next to its SW runtime channel with a
#   PASS/FAIL verdict, then live reading/volts/status per channel (exit non-zero on any failure).
#
# • min/max: From LoThreshRegVal / HiThreshRegVal (big-endian unsigned) × Scale;
#   fallback NormalMin / NormalMax, then WarningMin / WarningMax.
#
# • type: Per-channel file from JSON "Type". Typical values: rop, flex.
#
# • channel_name: Per-channel file from JSON ChnlNames array. Contains the human-readable
#   channel name (e.g. Chassis_0_LeakDetector_0_A). Located inside each channel directory.
#
# • device_name: Per-detector file from JSON "Name". Contains the human-readable detector
#   name (e.g. Chassis_0_LeakDetector_0_Mgmt).
#
# • hw-management-bmc-leakage-handler.sh (udev LEAKAGE<n> events) walks only numeric subdirs of
#   leakage/<n>/ (channel indices). It reads input, min, max; if sample is out of band it writes
#   last_sample and last_event.
#
# • Separate from this tree: init-time per-detector flags may appear under
#   /var/run/hw-management/system/leakage<n> (see helpers / bmc-ready integration).
